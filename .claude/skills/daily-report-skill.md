# 일일 업무 보고서 생성

인터뷰로 수집된 정보를 마크다운 보고서로 변환하고 파일로 저장합니다.

## 실행 절차

1. **형식 참고**: `Daily-Report/` 폴더의 최근 보고서 파일을 읽어 형식을 확인하세요
2. **보고서 작성**: 아래 형식에 맞춰 보고서를 작성하세요
3. **파일 저장**: `Daily-Report/daily_report_YYYYMMDD.md`로 저장하세요
4. **결과 표시**: 저장된 보고서 내용을 간략히 보여주세요

## 보고서 형식

```markdown
# 일일업무보고서 (YYYY-MM-DD)

- **작성자:** 김재구
- **작성일:** YYYY년 M월 D일 (요일)

---

## 금일 업무 내용

| 번호 | 업무 구분 | 업무 내용 | 비고 |
|:---:|:--------:|:--------|:---:|
| 1 | [구분] | [내용] | - |

---

## 명일 예정 업무

| 번호 | 업무 구분 | 업무 내용 |
|:---:|:--------:|:--------|
| 1 | [구분] | [내용] |

---

## 특이사항

> [특이사항 내용 또는 "특이사항 없음"]
```

## 업무 구분 기준

| 입력 예시 | 업무 구분 |
|---------|---------|
| 회의, 미팅, 협의 | 회의 |
| 세미나, 교육, 강의 | 교육 |
| 행사, 졸업식, 시상식 | 행사 |
| 도면, 설계, CAD | 설계/도면 |
| 개발, 코딩 | 개발 |
| 보고서, 문서, 기안 | 문서 |
| 점심, 식사, 잡무 | 기타 |

## 주의사항

- 날짜는 시스템의 currentDate를 사용하세요
- 파일명 형식: `daily_report_YYYYMMDD.md` (예: `daily_report_20260224.md`)
- 파일 저장 경로는 현재 프로젝트의 `Daily-Report/` 폴더입니다
