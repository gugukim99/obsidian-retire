# 일일 업무 보고서 작성

사용자와 짧은 대화를 통해 정보를 수집한 후, 마크다운 형식의 일일 업무 보고서를 생성하고 파일로 저장합니다.

## 절차

1. **오늘 업무 수집**: 사용자에게 오늘 수행한 업무를 물어보세요
2. **내일 예정 수집**: 내일 예정된 업무를 물어보세요
3. **특이사항 확인**: 특이사항이나 이슈가 있는지 확인하세요 (없으면 "없음" 처리)
4. **기존 보고서 참고**: `Daily-Report/` 폴더의 최근 보고서를 읽어 형식을 맞추세요
5. **보고서 저장**: `Daily-Report/daily_report_YYYYMMDD.md` 파일로 저장하세요

## 보고서 형식

```markdown
# 일일업무보고서 (YYYY-MM-DD)

- **작성자:** 김재구
- **작성일:** YYYY년 M월 D일 (요일)

---

## 금일 업무 내용

| 번호 | 업무 구분 | 업무 내용 | 비고 |
|:---:|:--------:|:--------|:---:|
| 1 | [구분] | [내용] | - |

---

## 명일 예정 업무

| 번호 | 업무 구분 | 업무 내용 |
|:---:|:--------:|:--------|
| 1 | [구분] | [내용] |

---

## 특이사항

> [특이사항 내용 또는 "특이사항 없음"]
```

## 업무 구분 예시

| 사용자 입력 | 업무 구분 |
|-----------|---------|
| 회의, 미팅 | 회의 |
| 세미나, 교육, 강의 | 교육 |
| 행사, 졸업식, 시상식 | 행사 |
| 도면, 설계 | 설계/도면 |
| 개발, 코딩 | 개발 |
| 보고서, 문서 | 문서 |
| 기타, 잡무 | 기타 |

## 주의사항

- 사용자의 짧은 입력(예: "회의함", "세미나들음")도 자연스럽게 해석해 표에 기재하세요
- 날짜는 오늘 날짜(시스템의 currentDate)를 사용하세요
- 파일명은 `daily_report_YYYYMMDD.md` 형식 (예: `daily_report_20260224.md`)
- 저장 후 생성된 보고서 내용을 간략히 보여주세요
